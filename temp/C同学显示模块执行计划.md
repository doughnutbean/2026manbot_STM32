# C同学显示模块执行计划（阶段一）

## 1. 背景与目标

依据当前项目文档与分工，C同学负责显示模块：消费 `MB_DisplayEvent_t`，完成显示驱动与表情渲染链路。

本轮目标（已确认）：
- 采用 I2C 方案（PB6/PB7）
- 型号未知，按 SSD1306 兼容最小方案实现
- 先完成最小表情集（HAPPY/SAD）占位
- 先打通“初始化 + 清屏 + 点亮/基础图样显示”

## 2. 约束

- 不改变任务/队列拓扑：保持 `Main -> Display` 队列模型
- 不修改 `Drivers/` 第三方库源码
- 不复用保留引脚：PB0（心跳）、PA9（日志）
- 避免 SPI 与 B 同学 `PA7` 冲突，优先 I2C

## 3. 执行步骤

1. 在 `App/Src/mb_tasks.c` 内新增显示硬件最小驱动函数（阶段一先内聚，减少工程文件改动）
2. 初始化 I2C1（PB6/PB7），实现 OLED 命令/数据发送
3. 实现 SSD1306 最小初始化序列
4. 实现 `clear` 与 `boot pattern`（上电可见）
5. 在 `MB_TaskDisplay` 中接入初始化逻辑
6. 在 `MB_TaskDisplay` 中接入最小表情映射：HAPPY/SAD
7. 保持其余表情/命令回退到通用占位图
8. 记录执行日志与结果

## 4. 验收标准（本轮）

- 代码结构不破坏现有任务框架
- `MB_TaskDisplay` 能完成初始化并显示基础图样
- 收到显示事件后，能切换至少 2 类占位图（HAPPY/SAD）
- 不引入 Drivers/Objects/Listings 非必要改动

## 5. 风险与预案

- OLED 实际型号非 SSD1306：预留后续切换 SH1106 初始化序列
- I2C 地址不匹配：默认 0x3C，后续可扩展 0x3D
- 硬件上拉或接线问题：通过“初始化后是否可见图样”快速定位
