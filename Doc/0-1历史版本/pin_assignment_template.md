# 硬件引脚分配表（阶段一模板）

> 维护人：成员 D  
> 目标：冻结资源冲突，统一 SPI / I2C / UART / DMA / 中断优先级

## 1. 外设总览

| 模块 | 功能器件 | 外设接口 | STM32 引脚 | DMA | 中断 | 负责人 | 状态 |
|---|---|---|---|---|---|---|---|
| 语音识别 | LD3320 / SU-03T | SPI1 / UARTx | 待定 | 待定 | 待定 | A | 待确认 |
| 语音播报 | SYN6288 | UARTx | 待定 | 可选 | 待定 | A | 待确认 |
| 舵机控制 | PCA9685 | I2C1 / I2C2 | 待定 | 无 | 可选 | B | 待确认 |
| 显示屏 | ILI9341 | SPI2(建议) + DMA | 待定 | 待定 | 待定 | C | 待确认 |
| 调试链路 | 串口调试/上位机 | USART1/2/3 | 待定 | 可选 | 待定 | E | 待确认 |

## 2. 建议约束（先验）

1. `SPI1` 优先给语音识别，`SPI2` 优先给 TFT（避免带宽互抢）。
2. `I2C1` 保留给 `PCA9685`，便于运动控制时序稳定。
3. 至少保留 1 路独立调试串口给 E（建议 `USART1`）。
4. 语音串口与调试串口不要复用同一 UART。
5. TFT 刷新 DMA 通道与高频串口 DMA 避免冲突。

## 3. FreeRTOS 任务优先级建议（初版）

| 任务 | 优先级建议 | 备注 |
|---|---:|---|
| MainTask | 高 | 状态机与分发核心 |
| MotionTask | 高 | 动作执行实时性要求高 |
| VoiceTask | 中高 | 指令入口，需低延迟 |
| DisplayTask | 中 | 可适当让步 |
| TtsTask | 中低 | 非强实时 |
| HeartbeatTask | 低 | 监控任务 |

## 4. 冻结流程

1. A/B/C/E 各自填充本模块外设需求。
2. D 汇总后检查冲突并出 `v1.0`。
3. 全员评审通过后冻结，不再随意改引脚。
